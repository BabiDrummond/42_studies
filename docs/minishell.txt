Builtins:
echo (com opção -n)
- Argumentos: Opções, String, Variáveis expandidas
- Retorno: Exibir string na tela
- Fluxo de execução:
    - Receber args
    - Verificar se opção -n é fornecida. Se sim, guardar numa variável booleana
    - Printar string e variáveis expandidas (convertidas para string anteriormente)
    - Se opção -n foi fornecida, não printar quebra de linha. Se não, printar quebra de linha

cd (caminho relativo ou absoluto)
- Argumentos: String [diretório]
- Retorno: Mudança de diretório
- Fluxo de execução:
    - Receber infos do envp e args
    - Checar valor do PWD do envp
    - Checar string para definir target dir
    - Se string .., deve-se remover um nível de diretório
    - Se string ~, deve-se mover para diretório /home
    - Se string contém nome de diretório, verificar se diretório existe e mover para ele. Caso não exista, informar que diretório não existe
    - Se houver uma / no meio da string, aplicar split na barra, e executar da esquerda para a direta (recursivamente)
    - Ir guardando o diretório atual em cada iteração
    - Ao final, dar unset na variável de ambiente PWD, e export no novo valor atualizado

pwd
- Argumentos: Nenhum
- Retorno: Caminho do diretório atual
- Fluxo de execução:
    - Receber infos do envp
    - Checar valor do PWD do envp
    - Printar na tela valor do PWD

export
- Argumentos: Envp, Variável {key, value}
- Retorno: Envp atualizado com nova variável
- Fluxo de execução:
    - Receber lista ou hashtable com valores do envp, nome da variável e valor da variável
    - Percorrer o envp para checar se há alguma variável de mesmo nome
    - Se sim, atualizar valor após nome da variável com valor fornecido na função
    - Se não, adicionar mais um nó na lista, contendo nome da variável e valor

unset
- Argumentos: Envp, Variável {key}
- Retorno: Envp atualizado com variável removida
- Fluxo de execução:
    - Receber lista ou hashtable com valores do envp e nome da variável
    - Percorrer o envp até encontrar variável de mesmo nome
    - Se encontrar, remover o nó da lista
    - Se não, retornar lista sem alterações

env
- Argumentos: Envp
- Retorno: Printar conteúdo do envp (todas as variáveis de ambiente salvas)
- Fluxo de execução:
    - Percorrer a lista do envp
    - Printar o valor de cada nó

Operadores lógicos:
&& e ||
Funcionam a partir do retorno da execução da função anterior.
cmd1 && cmd2 => Roda cmd2 APENAS se cmd1 retornou sucesso (0);
cmd1 || cmd2 => Roda cmd2 APENAS se cmd1 retornou erro (diferente de 0).

Redirecionamentos:
> => Muda o stdout para file descriptor especificado após o >. Se arquivo não existir, é criado. Se já existir, é truncado (apagado e sobrescrito).
fd = open("file.txt", O_WRONLY | O_CREAT | O_TRUNC, 0644);
>> => Muda o stdout para file descriptor especificado após o >>. Se arquivo não existir, é criado. Se já existir, é adicionado ao final (não é apagado).
fd = open("file.txt", O_WRONLY | O_CREAT | O_APPEND, 0644);
< => Muda o stdin de onde o comando está lendo. Exemplo, "cat < file.txt" vai ler do file.txt ao invés da entrada padrão (teclado).
<< => HEREDOC. Muda o stdin para pegar tudo DIGITADO entre os delimitadores fornecidos pelo usuário.

Pipe:
- Muda o stdin e o stdout, criando um canal que redireciona o stdout de um comando para ser o stdin do comando seguinte.
- Para isso, ele cria um canal (pipe), que conecta o stdout da esquerda, ao stdin da direita.
